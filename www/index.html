JSON BinPack

The Space-efficient Binary JSON Serialization Format

Characteristics

JSON BinPack is an open-source binary JSON serialization format with a strong focus on space efficiency. It supports schema-driven and schema-less modes to encode any JSON document given a matching JSON Schema definition.

Highly Space-efficient

A reproducible benchmark study has proven JSON BinPack to be space-efficient in comparison to 12 alternative binary serialization formats in every considered case.

Based on JSON Schema

JSON BinPack adopts the industry-standard schema language for JSON documents. Re-use the same schemas you already have in your OpenAPI, AsyncAPI or RAML definitions.

Optional Schema

JSON BinPack is a hybrid serialization format that runs in schema-driven or schema-less modes. You can be as loose or specific as your use case demands.

Open-source

The JSON BinPack encoding specification and its leading implementation are hosted on GitHub and released under the commercial-friendly Apache-2.0 open-source license.

How it works

JSON BinPack converts a JSON Schema definition and a matching JSON instance into a space-efficient binary representation that can be deserialized into the original input data without loss of information.

FAQ

What about data compression?

Our reproducible benchmark study shows that the semantic information provided by detailed schema definitions enables a serialization format such as JSON BinPack to often outperform general-purpose compressors configured with even their highest supported compression levels.

What about runtime-efficiency?

When transmitting data over the Internet, time is the bottleneck, making computation essentially free in comparison. JSON BinPack particularly excels at improving network performance when transmitting data through unreliable and low-bandwidth connections on mobile and IoT systems.

Where did this come from?

JSON BinPack is the result of extensive binary serialization research at University of Oxford. The project is led by Juan Cruz Viotti, a computer scientist with first-hand exposure at the problem of space-efficient network communication in the context of IoT and APIs.
